## <a name="about"></a>📖 نبذة عن المشروع

**MANSA** هي منصة تعليمية تفاعلية مصممة خصيصاً لطلاب السنة الأولى في كلية الحاسبات والمعلومات. تهدف المنصة إلى توفير بيئة تعليمية محفزة وممتعة من خلال:

- 🎯 **أسئلة تفاعلية** مع شرح مفصل
- ⏱️ **وضع التحدي** لاختبار المعرفة
- 🏆 **لوحة المتصدرون** للمنافسة الشريفة
- 🤖 **مساعد ذكاء اصطناعي** للإجابة على الاستفسارات
- 📱 **Progressive Web App** يعمل Offline

## <a name="quick-start"></a>🚀 البدء السريع

### المتطلبات

- [Node.js](https://nodejs.org) (الإصدار 18 أو أحدث)
- [Git](https://git-scm.com)
- حساب [Firebase](https://firebase.google.com) (مجاني)

### التثبيت

```bash
# 1. Clone المشروع
git clone https://github.com/username/mansa.git
cd mansa

# 2. تثبيت Dependencies
npm install

# 3. إعداد Environment Variables
cp .env.example .env.development

# 4. املأ المفاتيح في .env.development
# احصل على Firebase keys من: https://console.firebase.google.com
# احصل على Groq API key من: https://console.groq.com
# احصل على Gemini API key من: https://makersuite.google.com

# 5. تشغيل المشروع
npm run dev
```

**افتح المتصفح على:** `http://localhost:3000`

## <a name="architecture"></a>🏗️ البنية المعمارية

### نمط المعمارية

نستخدم **Feature-based Modular Architecture**:

```
┌─────────────────────────────────────┐
│         User Interface (HTML)        │
└─────────────┬───────────────────────┘
              │
┌─────────────▼───────────────────────┐
│      Page Controllers (JS)          │
│  ┌──────────┬──────────┬─────────┐ │
│  │  Home    │ Subject  │  Admin  │ │
│  └──────────┴──────────┴─────────┘ │
└─────────────┬───────────────────────┘
              │
┌─────────────▼───────────────────────┐
│         Feature Modules             │
│  ┌──────┬────────┬──────┬────────┐ │
│  │ Auth │ Quiz   │ Chat │ Leader │ │
│  └──────┴────────┴──────┴────────┘ │
└─────────────┬───────────────────────┘
              │
┌─────────────▼───────────────────────┐
│      Core Services & Utils          │
│  ┌──────────┬──────────┬─────────┐ │
│  │ Firebase │ i18n     │ State   │ │
│  └──────────┴──────────┴─────────┘ │
└─────────────────────────────────────┘
```

### Data Flow

```
User Action
    ↓
Event Handler (UI)
    ↓
Feature Module (Logic)
    ↓
Firebase Service (API)
    ↓
Firestore Database
    ↓
Update State
    ↓
Re-render UI
```

## <a name="pull-requests"></a>🔀 Pull Requests

### سير العمل

```bash
# 1. Fork المشروع
# اضغط "Fork" في GitHub

# 2. Clone الـ Fork
git clone https://github.com/YOUR-USERNAME/mansa.git
cd mansa

# 3. أضف remote للمشروع الأصلي
git remote add upstream https://github.com/username/mansa.git

# 4. أنشئ فرع جديد
git checkout develop
git pull upstream develop
git checkout -b feature/amazing-feature

# 5. اعمل على الكود
# ...

# 6. Commit
git add .
git commit -m "feat(scope): add amazing feature"

# 7. Push
git push origin feature/amazing-feature

# 8. افتح PR على GitHub
# اذهب لصفحة الـ fork واضغط "New Pull Request"
```

### متطلبات PR

قبل فتح PR، تأكد:

#### Code Quality
- [ ] الكود يعمل بدون أخطاء
- [ ] `npm run lint` بدون errors
- [ ] `npm run format` تم تشغيله
- [ ] لا warnings في Console

#### Testing
- [ ] اختبرت على Chrome
- [ ] اختبرت على Firefox
- [ ] اختبرت على Mobile (DevTools)
- [ ] اختبرت Offline (إن كان PWA)

#### Documentation
- [ ] أضفت/حدّثت JSDoc comments
- [ ] حدّثت README إن لزم
- [ ] حدّثت CHANGELOG.md

#### Git
- [ ] رسائل Commits واضحة
- [ ] فرع واحد لميزة واحدة
- [ ] Rebased على آخر develop

### قالب PR

عند فتح PR، املأ القالب:

```markdown
## الوصف
[وصف واضح للتغييرات]

## نوع التغيير
- [ ] Bug fix

- **Social Engineering** - خداع المستخدمين
- **DoS على Shared Hosting** - نستخدم Azure (محمي)
- **Clickjacking على صفحات عامة** - ليس فيها بيانات حساسة
