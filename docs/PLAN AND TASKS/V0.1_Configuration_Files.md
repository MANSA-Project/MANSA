### المهمة 1.4: قالب Commit Message

**المسار:** `.gitmessage`

**الغرض:**  
قالب لرسائل الـ commits لتشجيع Conventional Commits.

**الكود:**

```
# <type>(<scope>): <subject>
# |<----  max 50 chars  ---->|

# Body - explain WHY not WHAT (optional)
# |<----  max 72 chars  ---->|

# Footer (optional)
# - BREAKING CHANGE: <description>
# - Closes #<issue number>
# - Refs #<issue number>

# ============================================
# Types:
#   feat:     New feature
#   fix:      Bug fix
#   docs:     Documentation changes
#   style:    Code style (formatting, no logic change)
#   refactor: Code restructuring (no feature/fix)
#   perf:     Performance improvement
#   test:     Adding/updating tests
#   chore:    Maintenance tasks
#   ci:       CI/CD changes
#   build:    Build system changes
#   revert:   Revert previous commit
# ============================================
# Scope (optional): auth, quiz, ui, api, db, etc.
# ============================================
# Subject:
#   - Use imperative mood ("add" not "added")
#   - Don't capitalize first letter
#   - No period at the end
# ============================================
# Examples:
#   feat(auth): add email verification
#   fix(quiz): resolve timer not stopping
#   docs(readme): update installation steps
#   refactor(api): simplify error handling
#   perf(images): add lazy loading
# ============================================
```

**تفعيل القالب:**

```bash
# تشغيل مرة واحدة في المشروع
git config commit.template .gitmessage
```

**ملاحظات:**
- 📝 يظهر القالب تلقائياً عند كتابة commit
- ✅ يساعد في الالتزام بـ Conventional Commits
- 🎯 أمثلة واضحة لكل نوع
